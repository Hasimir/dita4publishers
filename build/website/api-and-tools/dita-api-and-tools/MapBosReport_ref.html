MapBosReporter

Generates a DITA map tree or full map bounded object set (BOS) report.

MapBosReport -i input map file [ -o report file ] [ -c XML catalog file ] [ -m ] [ -d Ditaval file ]
Parameters
Option Value Description
-i Path and filename of the root map to process  
-o (optional) Path and filename of the generated BOS report. If not specified, report is written to STDOUT.  
-c (optional) Path and filename of the XML catalog to use for resolving DTD and schema references. If not specified, and the DITA_HOME environment variable is set, the catalog-dita.xml file will be used, if found. If DITA_HOME is not set, then no catalogs are used.  
-m (optional) When specified, only the map tree, not the full BOS, is constructed and reported.  
-d (optional) Path and filename of the Ditaval file to use when determining the key space and constructing the BOS. If not specified, all elements are applicable.  

Usage

To run from the command line, use a Java command like so (all on one line):
java -Xmx512M -jar dita4publishers-mapbosreporter.jar 
  -i ~/workspace/dita_12_spec/dita1.2-complete.ditamap  
  -o bos-report.txt

As the BOS construction process builds DOMs of each document in the BOS, larger maps will require more Java memory, as specified by the -Xmx Java parameter. This example processes the DITA 1.2 full specification (architectural spec and language reference), consistent of 779 separate documents.

The resulting report consists of the BOS report, listing the map tree and all the topics used from it (representing topics as children of maps) and all the dependencies of each topic (other topics or non-DITA resources linked to from the topic), followed by the key space report, listing all the key definitions in the map tree. If you specify -m, the BOS report reflects only the map tree, no topics.
