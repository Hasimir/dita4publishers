DitaDxpMapPackager

Creates a DITA Interchange Package (DXP) Zip file from a DITA map containing all the local dependencies required by the map.

DitaDxpMapPackager -i input map file [ -o DXP zip file ] [ -c XML catalog file ] [ -d Ditaval file ]
Parameters
Option Value Description
-i Path and filename of the root map to process  
-o (optional) Path and filename of the DXP package. If not specified, a package named mapFilename.dxp in the same directory as the input map will be created.  
-c (optional) Path and filename of the XML catalog to use for resolving DTD and schema references. If not specified, and the DITA_HOME environment variable is set, the catalog-dita.xml file will be used, if found. If DITA_HOME is not set, then no catalogs are used.  
-d (optional) Path and filename of the Ditaval file to use when determining the key space and constructing the BOS. If not specified, all elements are applicable.  

Usage

To run from the command line, use a Java command like so (all on one line):
java -Xmx512M -jar dita4publishers-dxppackager.jar 
  -i ~/workspace/dita_12_spec/dita1.2-complete.ditamap  
  -o ditaspec-complete.dxp

As the BOS construction process builds DOMs of each document in the BOS, larger maps will require more Java memory, as specified by the -Xmx Java parameter. This example processes the DITA 1.2 full specification (architectural spec and language reference), consistent of 779 separate documents.

The resulting Zip file contains all the local dependencies required by the root map as determined by the BOS construction process.

The DITA Interchange Package specification requires that the resulting package have either exactly one root map in the top-level directory or have a DXP manifest map. By default the DXP map packager reorganizes the BOS members so that the root map is by itself in the root directory and all dependencies are in directories below it. If files have to be moved to accomplish this all affected pointers will be rewritten to reflect the moves.
