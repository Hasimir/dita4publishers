Using the DITA For Publishers Source

You can also use the DITA For Publishers materials directly from the project's source code control system. You can also use the materials as a model for local development setups for developing your own DITA specializations. The materials are managed using Subversion through SourceForge. You can use any Subversion client to access the source code, such as TortoisSVN, the Oxygen Subversion client, or the Subclipse plugin for Eclipse.

  1. Using a Subversion client, check out a read-only copy of the materials, following the instructions on the project's code access page: https://sourceforge.net/p/dita4publishers/code
  2. Create a file named build.properties or .build.properties, either in the project's directory or in your home directory.

    This file is used by the project's Ant build script to configure the location of the DITA Open Toolkit.

  3. Edit the build.properties file you just created and add the property "dita-ot-dir".
    The property specification should look like this:
    dita-ot-dir={path-to-toolkit}

    Where "{path-to-toolkit} is wherever directory contains your Toolkit, e.g. "c:/dita-ot" or "/Applications/Oxygen/frameworks/dita/DITA-OT" or whatever.

    If you use Oxygen or similar tool that provides a Toolkit and maintain multiple versions of the tool, it can be useful to define a separate property for the editor location, e.g.:
    oxygen_home=/Applications/oxygen_10.2
dita-ot-dir=${oxygen_home}/frameworks/dita/DITA-OT
  4. Save the file.
  5. Test the DITA for Publishers' Ant file by running the target "deploy-toolkit-plugins".

    From a command line where the dita4publishers directory is the current directory, run this command (assuming Ant in your PATH):

    ant deploy-toolkit-plugins

    You should see messages indicating that the plugins have been copied to your toolkit and the integrate task has been run.

You should now be ready to use the Publishing specializations with the Toolkit and edit documents that use them using any XML editor that uses your Toolkit to get the DTDs its needs (e.g., OxygenXML).

Other editors may require more work to configure to enable editing.
